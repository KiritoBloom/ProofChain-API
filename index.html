Integrity flow

How the system proves tamper evidence.

ProofChain is designed so verification can happen outside the server. The public API exposes the exact artifacts needed to recompute membership, validate signatures, and inspect anchor continuity.

01

Ingest

Clients post immutable event envelopes and receive deterministic SHA-256 hashes.

02

Seal

Pending events are ordered into blocks, rooted with a Merkle tree, and signed with Ed25519.

03

Anchor

Sealed blocks are chained into append-only transparency checkpoints for public inspection.

04

Verify

Any client can fetch a proof envelope and verify membership, signature, and anchor integrity.
