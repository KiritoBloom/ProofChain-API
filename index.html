03 — Integrity flow

How the system proves tamper evidence.

Verification can happen entirely outside the server. The public API exposes every artifact needed to recompute membership, validate signatures, and inspect anchor continuity.

1

Ingest

Clients post immutable event envelopes. Each receives a deterministic SHA-256 hash computed from canonical JSON.

2

Seal

Pending events are ordered into blocks, rooted with a Merkle tree, and signed with Ed25519 using a versioned key.

3

Anchor

Sealed blocks are chained into append-only transparency checkpoints available for public inspection.

4

Verify

Any client can fetch a proof envelope and independently verify membership, signature, and anchor integrity.
